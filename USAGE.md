# 使用说明

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

**注意**: 
- 如果有NVIDIA GPU，建议安装支持CUDA的PyTorch版本以获得更好的性能
- 访问 https://pytorch.org/ 获取适合您系统的安装命令

### 2. 运行程序

```bash
python main.py
```

### 3. 使用步骤

1. **选择文件夹**: 点击"浏览"按钮选择要扫描的视频文件夹
2. **调整设置**（可选）: 点击"设置"按钮调整检测参数
3. **开始检测**: 点击"开始检测"按钮
4. **查看结果**: 检测完成后，重复视频会分组显示
5. **处理重复项**: 
   - 绿色高亮的是推荐保留的版本（最高质量）
   - 勾选要删除的视频
   - 点击"删除选中"执行删除
   - 或点击"导出报告"保存结果

## 高级设置说明

### 相似度阈值 (0.70-0.95)
- **值越大越严格**: 只有内容几乎完全相同的视频才算重复
- **值越小越宽松**: 内容相似的视频也算重复
- **推荐值**: 0.85（平衡模式）

### 帧采样率
- 每秒提取多少帧用于CLIP分析
- 值越大越精确但越慢
- **推荐值**: 2.0（每秒2帧）

### 批处理大小
- GPU一次处理的帧数
- 显存不足时可降低此值（如16或8）
- 显存充足时可提高此值（如64）

## 预设模式

### 快速模式
- 相似度阈值: 0.80（宽松）
- 采样率: 1.5帧/秒
- 适合快速清理，容忍度较高

### 平衡模式（推荐）
- 相似度阈值: 0.85
- 采样率: 2.0帧/秒
- 速度和准确度的平衡

### 精确模式
- 相似度阈值: 0.90（严格）
- 采样率: 3.0帧/秒
- 最高准确度，但速度较慢

## 性能优化建议

### 如果检测速度慢
1. 降低帧采样率（从2.0降到1.5）
2. 启用GPU加速（需要NVIDIA显卡）
3. 减少批处理大小（如果显存不足）
4. 使用快速模式预设

### 如果内存不足
1. 降低帧采样率
2. 减少批处理大小
3. 分批处理视频（分多次扫描不同文件夹）

### 如果显存不足
1. 降低批处理大小（从32降到16或8）
2. 降低采样率
3. 关闭其他占用GPU的程序

## 常见问题

### Q: 首次运行很慢？
A: 首次运行会下载CLIP模型（约400MB），需要网络连接和一些时间。模型下载后会缓存，后续运行会快很多。

### Q: 没有NVIDIA GPU怎么办？
A: 程序会自动使用CPU模式，功能完全相同，只是速度会慢一些。

### Q: 检测结果不准确？
A: 可以调整相似度阈值。如果误报太多，提高阈值（更严格）；如果漏报太多，降低阈值（更宽松）。

### Q: 可以恢复删除的视频吗？
A: 不能！删除是永久性的。建议先导出报告检查，或者先移动到回收站而不是直接删除。

### Q: 支持哪些视频格式？
A: 支持常见格式：mp4, avi, mkv, mov, flv, wmv, webm, m4v, mpg, mpeg

### Q: 缓存文件在哪里？
A: 缓存数据库位于 `cache/video_cache.db`，可以在设置中清理。

## 技术原理

本软件采用智能两阶段检测策略：

1. **第一阶段 - 元数据快速分组**: 
   - 使用视频时长、分辨率、帧数等元数据创建指纹
   - 只有元数据相近的视频才可能是重复的
   - 速度极快（比传统哈希快100倍以上）
   - 可选：使用文件部分哈希快速识别完全相同的副本

2. **第二阶段 - CLIP深度验证**:
   - 使用CLIP深度学习模型提取视频特征
   - 提取512维特征向量
   - 通过余弦相似度精确判断是否重复
   - 对压缩、格式转换等变化具有鲁棒性

这种策略比传统感知哈希快10-50倍，准确度更高。

## 打包为可执行文件

如果要分发给没有Python环境的用户：

```bash
pip install pyinstaller
python build.py
```

生成的可执行文件位于 `dist/视频重复检测工具/` 目录。
